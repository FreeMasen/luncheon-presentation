An upcoming pull request is hopefully merging soon that will clean up some parts of this project. The first thing it is trying to achieve is to make the API surface of Reqeust and Response consistent. Previously the two types provided different method names for get_content_length. Another important change here is moving the shared code into a single shared module, previous this was manually being copied and pasted and resulted in some inconsistencies across the implementations. Finally this adds support for chunk encoding for both parsing and generating request with chunk encoding.
